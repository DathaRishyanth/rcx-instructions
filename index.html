Quick Overview

This document covers how to attempt the 2 server-based reverse coding (named P1 and P2).

  • Connect to the Reverse Coding server from your terminal (Windows / Linux / Mac / Google Colab).
  • Select a problem, provide input, and the server returns expected output.
  • Implement the same logic in HackerEarth code editor,(as per the input format specified in HackerEarth) and submit your solutions.

§0 Before You Begin

  • Ensure you have a working terminal (if not you can use google colab we set up) and internet connection.
  • Join the WhatsApp grp for querries. We'll do dry run of the server login on 11th Dec.
  • Login to your HackerEarth accountc and be ready the contest group.

§1 Windows Users

Windows Quick Connect (Telnet)

  1. Open Command Prompt as Administrator.

  2. Enable Telnet:

    dism /online /Enable-Feature /FeatureName:TelnetClient

  3. Verify by typing:

    telnet

  4. Connect to the server:

    open <ip> <port>

    Replace <ip> and <port> with the values provided in the contest.

    :: Example 
open 157.0.1.1 6060

§2 Linux Users

Linux quick connect (Netcat)

  1. Check if nc is installed:
    nc -h
  2. If not installed, install it:
    • Debian / Ubuntu:
    sudo apt update
sudo apt install netcat
    • Arch
    sudo pacman-S openbsd-netcat
  3. Connect:
    nc -v <ip> <port>
    Replace <ip> and <port> with the values provided in the contest.
    # Example 
nc -v 157.0.1.1 6060

§3 Mac Users

Recent macOS versions ship with nc. If you face terminal permission or networking restrictions, use the Colab notebook (link below).
